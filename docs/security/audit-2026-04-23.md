# Collab 安全审计报告

日期：2026-04-23 | 审计人：飞马 | 触发：建军要求全面安全 review

## 审计范围

- `packages/server/src/` — 所有路由、认证、WS 端点
- `packages/plugin/src/` — Plugin 客户端代码
- 重点：凭证传递方式、Cookie 安全、输入校验、权限检查

## 发现

### P0 — 凭证通过 query string 传递（4 处）

URL query string 会被日志、CDN、浏览器历史记录，不应传递敏感信息。

| Bug ID | 文件 | 行号 | 问题 | 修复方案 |
|--------|------|------|------|----------|
| COL-BUG-006 | `routes/ws-plugin.ts` | 17 | `url.searchParams.get('apiKey')` | 改读 `Authorization: Bearer` header |
| COL-BUG-007 | `ws.ts` | 26 | `url.searchParams.get('token')` | 改读 `Authorization: Bearer` header |
| COL-BUG-008 | `routes/ws-remote.ts` | 16 | `url.searchParams.get('token')` | 改读 `Authorization: Bearer` header |
| COL-BUG-009 | `routes/stream.ts` | 120 | `request.query.api_key` | 改读 `Authorization: Bearer` header |

**注意**：HTTP API 路由的认证（`auth.ts:74`）已正确使用 `Authorization: Bearer` header，只有 WS 和 SSE 端点有问题。

### P1 — Dev 模式认证绕过

| Bug ID | 文件 | 行号 | 问题 | 风险 |
|--------|------|------|------|------|
| COL-BUG-010 | `ws.ts` | 54 | `NODE_ENV=development` 时可通过 `user_id` query param 直接认证，无参数默认用第一个 admin | 如果生产环境误设 NODE_ENV=development 则完全绕过认证 |

**缓解**：确认生产 Dockerfile 和 compose 中 `NODE_ENV=production` 已设置。长期应移除此绕过逻辑或加 explicit dev flag。

### ✅ Cookie 安全 — 已正确配置

`auth.ts` 中所有 Set-Cookie 调用（3 处：登录、注册、登出）均设置：
- `HttpOnly` ✅ — 防 XSS 窃取
- `SameSite=Lax` ✅ — 防 CSRF
- `Secure`（生产环境条件启用）✅
- `Max-Age=7d` ✅

### ✅ 无硬编码 Secret

JWT_SECRET 从环境变量读取，无硬编码密钥。

### ✅ SQL 注入防护

所有数据库操作使用 `better-sqlite3` 的 prepared statements（`db.prepare().run/get/all`），无字符串拼接 SQL。

### 待确认

- XSS：用户消息内容的 escape 由前端处理（`packages/client`），server 端不做 HTML escape（API 返回原始内容）。需确认前端是否正确 escape。
- CORS：未检查到显式 CORS 配置，依赖 Caddy 反代的默认行为。
