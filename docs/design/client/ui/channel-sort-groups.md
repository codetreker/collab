# 频道排序与分组 — UI 线框图

> B26（频道拖动排序）+ B27（频道自定义分组）的 UI 设计。
> 桌面端宽度 ≈ 80 字符，移动端 ≈ 40 字符。
> 设计符号说明见 [README.md](README.md)。
>
> **核心原则：排序/分组是频道属性，由频道 owner 管理。**
> 拖动手柄（≡）仅在当前用户是 owner 的频道上显示。
> 非 owner 用户看到的是只读视图——没有手柄，不能拖动。

---

## 1. Sidebar 分组展示（桌面端 — Owner 视角）

```
+────────────────────+
│ COLLAB         [+] │  ← [+] 频道 owner：创建频道 / 创建分组
│                    │
│ ≡ # general        │  ← ≡ 拖动手柄（仅 owner 可见）
│ ≡ # random         │
│                    │
│ ▾ 🏗 工程          │  ← 分组 header：折叠箭头 + 分组名
│ ≡ # dev            │     owner 可拖动组内频道
│ ≡ # ci-cd          │
│ ≡ # infra          │
│                    │
│ ▾ 📐 设计          │
│ ≡ # design         │
│ ≡ # ux-research    │
│                    │
│ ▸ 📋 已归档        │  ← 折叠状态：▸ + 分组名，频道隐藏
│                    │
│ ▾ DIRECT MESSAGES  │
│   🟢 Bob           │
│   🟡 Carol         │
│                    │
│ [⚙][🤖][📁][🌐]   │
+────────────────────+
```

**说明（Owner 视角）：**
- 拖动手柄 `≡` 出现在当前用户创建的频道左侧
- Owner 可拖动自己的频道调整排序、拖入/拖出分组
- 分组 header 右键可重命名/删除（仅分组创建者）
- 未分组频道（`# general`、`# random`）显示在分组列表上方，无 header
- DM 列表不受分组影响，始终在频道列表下方

│   # ci-cd          │
│   # infra          │
│                    │
│ ▾ 📐 设计          │
│   # design         │
│   # ux-research    │
│                    │
│ ▸ 📋 已归档        │
│                    │
│ ▾ DIRECT MESSAGES  │

## 2. 拖动排序（桌面端 — Owner 拖动中状态）

```
+────────────────────+
│ COLLAB         [+] │
│                    │
│ ≡ # general        │
│ ≡ # random         │
│                    │
│ ▾ 🏗 工程          │
│ ≡ # dev            │
│ ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌ │  ← 蓝色插入指示线（目标位置）
│ ≡ # ci-cd          │
│ ≡ # infra          │
│                    │
│ ▾ 📐 设计          │
│ ┏━━━━━━━━━━━━━━━━┓ │
│ ┃  # ux-research ┃ │  ← 拖影：半透明频道项，跟随光标
│ ┗━━━━━━━━━━━━━━━━┛ │
│ ≡ # design         │     原位置显示为虚线占位
│   ┈ ┈ ┈ ┈ ┈ ┈ ┈  │  ← 原位占位符（虚线框）
│                    │
│ ▸ 📋 已归档        │
│                    │
│ ▾ DIRECT MESSAGES  │
│   🟢 Bob           │
+────────────────────+
```

**交互细节：**
- 鼠标按住拖动手柄（≡）即可拖动（只有 owner 的频道有手柄）
- 拖影半透明（opacity ≈ 0.6），跟随光标移动
- 蓝色插入线（2px，`var(--accent-color)`）标记目标插入位置
- 原位置显示虚线占位符
- 拖动到分组 header 上悬停 > 500ms → 展开该分组（若已折叠）
- 拖动到分组区域内松手 → 频道归入该分组
- 拖动到未分组区域松手 → 频道脱离分组
- 分组 header 之间也可拖动排序（整组移动，仅分组创建者可拖）
- **只有频道 owner 能拖动**——非 owner 用户无拖动手柄

## 3. 创建分组弹窗

```
+────────────────────────────────────+
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
│░░░ ┌─────────────────────────┐ ░░░│
│░░░ │  创建频道分组        [✕] │ ░░░│
│░░░ ├─────────────────────────┤ ░░░│
│░░░ │                         │ ░░░│
│░░░ │  分组名称               │ ░░░│
│░░░ │  ┌───────────────────┐  │ ░░░│
│░░░ │  │ 例如：工程         │  │ ░░░│
│░░░ │  └───────────────────┘  │ ░░░│
│░░░ │                         │ ░░░│
│░░░ │        [取消] [创建]    │ ░░░│
│░░░ │                         │ ░░░│
│░░░ └─────────────────────────┘ ░░░│
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
+────────────────────────────────────+
```

**触发方式：**
- 频道 owner 点击 sidebar 顶部 `[+]` 按钮 → 下拉菜单选 "创建分组"
- 或频道 owner 右键 sidebar 空白区域 → "创建分组"

**说明：**
- 输入框带 placeholder 提示
- "创建" 按钮在分组名非空时启用
- 创建成功后弹窗关闭，新分组出现在 sidebar 末尾（分组间）
- 创建者自动成为该分组的管理者

## 4. 分组管理（右键菜单 + 重命名 + 删除确认）

### 4a. 右键上下文菜单（仅分组创建者可见）

```
+────────────────────+
│ ▾ 🏗 工程          │
│ ≡ # dev     ┌─────────────┐
│ ≡ # ci-cd   │ ✏️ 重命名     │
│ ≡ # infra   │ 🗑 删除分组   │
│             └─────────────┘
│ ▾ 📐 设计          │
```

**触发：** 分组创建者右键点击分组 header → 弹出上下文菜单。
**非创建者右键：** 无管理选项（或不显示右键菜单）。

### 4b. Inline 重命名（仅分组创建者）

```
+────────────────────+
│ ▾ ┌──────────────┐ │
│   │ 🏗 工程       │ │  ← 分组名变为可编辑 input
│   └──────────────┘ │     Enter 确认 / Esc 取消
│ ≡ # dev            │
│ ≡ # ci-cd          │
```

### 4c. 删除分组确认弹窗（仅分组创建者）

```
+────────────────────────────────────+
│░░░ ┌─────────────────────────┐ ░░░│
│░░░ │  删除分组            [✕] │ ░░░│
│░░░ ├─────────────────────────┤ ░░░│
│░░░ │                         │ ░░░│
│░░░ │  确定删除分组 "工程"？   │ ░░░│
│░░░ │  组内 3 个频道将移到     │ ░░░│
│░░░ │  "未分组" 区域。        │ ░░░│
│░░░ │                         │ ░░░│
│░░░ │       [取消] [🗑 删除]   │ ░░░│
│░░░ └─────────────────────────┘ ░░░│
+────────────────────────────────────+
```

**说明：**
- 删除按钮红色（`btn-danger`）
- 删除后组内频道自动回到 "未分组" 区域
- 频道本身不受影响（只是失去分组归属）

**移动端交互细节：**
- 长按频道项的拖动手柄 ≥ 300ms → 触发震动反馈（haptic） + 进入拖动模式
- 拖影略微放大（scale: 1.03）并带阴影，增强 "提起" 感
- 手指移动时拖影跟随，蓝色插入线随目标位置移动
- 松手放下，排序即时生效，WS 广播给所有在线用户
- **只有频道 owner 能触发拖动**——非 owner 的频道无拖动手柄

## 6. 状态汇总

| 状态 | Owner 视图 | 非 Owner 视图 |
|---|---|---|
| 默认 | 拖动手柄可见，分组可右键管理，可折叠/展开 | 无手柄（只读），分组可折叠/展开 |
| 拖动中 | 拖影 + 插入线 + 原位占位符 | — （无法触发拖动） |
| 创建分组 | 弹窗输入名称 | — （无创建入口） |
| 重命名分组 | Inline 编辑 | — （无此选项） |
| 删除分组 | 确认弹窗 | — （无此选项） |
| 折叠/展开 | 点击 ▾/▸ 切换 | 点击 ▾/▸ 切换（相同） |

> 排序和分组是频道属性，由频道 owner 管理。所有用户看到相同的排序结果。折叠/展开是唯一的个人偏好（存 localStorage）。
