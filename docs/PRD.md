# Collab — PRD v2

日期：2026-04-18 | 状态：Draft

## 背景

团队当前使用 Discord 作为日常沟通工具，但老板（建军）在中国大陆，Discord 连接极不稳定，严重影响工作效率和沟通时效。团队需要一个自己部署、中国可直接访问的团队通讯平台。

Collab 的产品定位是**围绕内容的团队协作平台**。但"协作"的基础是"沟通"——没有稳定的日常通讯，协作无从谈起。因此产品策略调整为：

- **v1：轻量团队聊天** — 先解决日常通讯的刚需，替代 Discord
- **v2：画布 + 文档协作** — 在稳定通讯的基础上，叠加无限画布、文档卡片、AI 协作助手

## 目标用户

| 用户 | 特征 | 核心诉求 |
|------|------|----------|
| 建军（老板） | 在中国大陆，需要稳定连接 | 随时能和团队沟通，不受网络限制 |
| AI agent 团队 | 飞马/野马/战马/烈马，通过 OpenClaw 运行 | 原生接入聊天频道，与人类成员体验一致 |
| 团队成员 | 3-10 人小型技术团队 | 简单好用的团队聊天，手机电脑都能用 |

## 核心需求

### 需求 1: 频道系统

- 用户故事：作为团队成员，我想创建和切换不同的聊天频道（如 #general、#project-collab、#daily-standup），以便按话题组织讨论，避免信息混杂
- 验收标准：
  - [ ] 能创建新频道，指定频道名称
  - [ ] 侧边栏展示频道列表
  - [ ] 点击频道名可切换到对应频道
  - [ ] 每个频道的消息独立，互不干扰

### 需求 2: 实时消息

- 用户故事：作为团队成员，我想发送消息后所有在线成员立即看到，以便实时讨论工作
- 验收标准：
  - [ ] 通过 WebSocket 实现实时消息推送
  - [ ] 发送消息后，同频道所有在线成员即时收到（延迟 < 1s）
  - [ ] 消息支持纯文字和 Markdown 格式渲染（加粗、斜体、代码块、链接、列表）
  - [ ] 消息支持发送和显示图片（内联显示）
  - [ ] 消息显示发送者名称和时间戳
  - [ ] 网络断开时显示断连状态提示（如顶部 banner）
  - [ ] 断线后自动重连，重连成功后同步错过的消息
  - [ ] 连接状态可见：在线 / 断连 / 重连中

### 需求 3: @ Mention

- 用户故事：作为团队成员，我想在消息中 @ 某个人，以便明确通知对方关注这条消息
- 验收标准：
  - [ ] 输入 `@` 时弹出成员选择列表
  - [ ] 可通过输入名称过滤成员列表
  - [ ] 选择后在消息中插入 mention 标记
  - [ ] 被 mention 的成员看到该消息有视觉高亮提示
  - [ ] mention 渲染为可识别的样式（如高亮背景色）

### 需求 4: 消息历史

- 用户故事：作为团队成员，我想看到频道里之前的消息记录，以便了解之前讨论的上下文
- 验收标准：
  - [ ] 消息在服务端持久化存储
  - [ ] 进入频道时自动加载最近的消息
  - [ ] 刷新页面后历史消息仍在
  - [ ] 向上滚动可加载更早的消息（分页加载）
  - [ ] 加载过程中有明确的加载状态提示

### 需求 5: OpenClaw Channel Plugin

- 用户故事：作为 AI agent（飞马/野马/战马/烈马），我想通过 OpenClaw channel plugin 原生接入 Collab 频道，以便和人类成员在同一个频道里协作沟通
- 验收标准：
  - [ ] OpenClaw agent 能通过 channel plugin 发送消息到指定 Collab 频道
  - [ ] OpenClaw agent 能通过 channel plugin 接收 Collab 频道的新消息
  - [ ] agent 发送的消息在网页端显示格式与人类用户发送的一致
  - [ ] 支持 agent 发送 Markdown 格式消息
  - [ ] agent 的消息显示对应的 agent 名称（如"飞马"）
  - [ ] agent 接入需要认证机制（API key 或 token），防止未授权访问

### 需求 6: 网页端 + 移动端适配

- 用户故事：作为建军，我想在手机浏览器上也能正常使用 Collab，以便随时随地参与团队沟通
- 验收标准：
  - [ ] 纯网页端，无需安装任何 APP
  - [ ] 响应式布局，手机浏览器可正常使用
  - [ ] 移动端频道列表可折叠/展开，不遮挡消息区域
  - [ ] 移动端消息输入、发送、滚动体验流畅
  - [ ] 桌面端和移动端功能一致，无功能缺失

### 需求 7: 中国可访问

- 用户故事：作为建军，我想从中国大陆网络直接访问 Collab，以便不再依赖不稳定的 VPN 或代理
- 验收标准：
  - [ ] 部署在团队自有基础设施上
  - [ ] 不依赖任何被中国大陆防火墙封锁的外部服务（如 Google Fonts、CDN 等）
  - [ ] 从中国大陆网络可正常访问，页面加载时间 < 5s
  - [ ] WebSocket 连接在中国网络环境下稳定

## 成功指标

**定性目标：**
- 建军的日常团队沟通自然从 Discord 迁移到 Collab，不再受连接不稳定困扰
- 团队成员（包括 AI agent）感受不到工具切换带来的摩擦

**定量参考：**
- 日常消息 80%+ 发生在 Collab 上（而非 Discord）
- 建军在中国的消息收发延迟 < 1s（当前 Discord 经常断连）

## 开放问题

1. ~~**部署位置**~~ — ✅ 已解决。v1 先用 Cloudflare（在中国偶尔不稳定但可用，不像 Discord 完全不能用）。如果 Cloudflare 太不稳定，后续考虑迁移到国内机器。
2. ~~**与 Discord 的过渡期**~~ — ✅ 已解决。上线后先拿一个项目迁移过去做测试，解决阻塞问题后全量迁移。聊天记录迁移不在 v1 范围。
3. **RFR 迁移路径** — Ready-for-Review 已有的文档渲染能力，在 v2 整合画布和文档时如何复用？
4. **OpenClaw channel plugin 技术可行性** — 需要先调研 OpenClaw 的 channel plugin 架构（消息协议、webhook/WebSocket 对接方式），评估开发工作量。这是 v1 最大的技术风险，建议在开发前完成技术 spike。

## v2 路线图预告

以下方向已在团队讨论中确认，等 v1 稳定运行后启动：

- **无限画布 + Markdown 文档卡片** — 在画布上创建和组织文档，支持自由布局
- **浮动对话锚定到文档** — 讨论可以关联到具体文档或文档片段，实现"围绕内容的协作"
- **AI 协作助手** — AI agent 可以对文档提出建议（suggestion），人类成员可以 accept / reject

这些功能将实现 Collab 的完整产品愿景：从团队聊天进化为**围绕内容的团队协作平台**。
