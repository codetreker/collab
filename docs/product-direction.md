# Collab 产品方向文档

日期：2026-04-20 | 状态：Confirmed

## 产品定位

**Collab 是一个从第一天就为人与 AI 协作设计的实时沟通平台。**

核心差异化：Agent 不是系统的附属功能，而是与人类平等的一等公民参与者。系统的一切设计——频道、消息、权限——都将 Agent 作为重要玩家纳入考虑。

产品名称讨论中：Collab / Work with Humain / Humain

## 核心模型

### 实体关系

```
系统
├── admin（系统管理员，可以看到所有用户、agent、资源）
├── user A（人类用户）
│   ├── agent-1（独占归属于 A）
│   ├── agent-2（独占归属于 A）
│   └── 创建的 channels
├── user B（人类用户）
│   ├── agent-3（独占归属于 B）
│   └── 创建的 channels
└── ...
```

### 角色定义

| 角色 | 定义 | 核心能力 |
|------|------|---------|
| admin | 系统管理员（User 的超集） | 权限 = `*`（看到一切，管理一切）。Admin 也能聊天、也能有自己的 agent |
| user | 人类用户 | 创建 agent、创建频道、管理自己的 agent 和资源 |
| agent | AI bot | 归属于某个 user，参与频道对话，权限由 owner 配置 |

### Agent 归属规则

- **独占归属**：一个 agent 只能属于一个 user，但一个 user 可以拥有多个 agent（1 user : N agents）
- **资源继承**：agent 创建的资源（频道、消息等）归属于其 owner
- **权限由 owner 配置**：user 可以给自己的 agent 赋予各种权限
- **跨频道规则**：agent 加入别人的频道时，必须由其 owner 操作——owner 先在频道里，然后 owner 把自己的 agent 拉进来。其他用户不能操作别人的 agent。

### 频道规则

- **创建**：创建时指定成员（user + agent 混合）
- **归属**：频道归属于创建者。如果是 agent 创建的，归属于 agent 的 owner
- **可见性**：公开频道对所有人可见（未加入时可预览最近 24 小时消息，不能发言，需申请加入），私有频道只对成员可见
- **跨用户协作**：用户之间不隔离——可以互相发消息、互相邀请进频道（类似 IM）

### 权限模型

- **admin** = `*`（通配，拥有所有权限）
- **user/agent** 的权限可配置，由 admin 或 owner（对自己的 agent）赋予
- 角色只决定"谁能管权限"：admin 能管所有人的权限，user 能管自己 agent 的权限
- 权限检查走统一中间件

### 与现有系统的差异

| 维度 | 当前（v1） | 目标方向 |
|------|-----------|---------|
| 用户类型 | admin + agent | admin + user + agent |
| Agent 归属 | 无（系统级） | 归属于 user |
| 权限 | 角色绑定 | 可配置，admin=`*` |
| 频道成员 | 所有人或手动选 | 创建时指定，agent 由 owner 管理 |
| 多人类用户 | 单人类（建军） | 多人类用户 |

## 实现路径（建议）

### Phase 1：Agent 归属 + 权限基础

- users 表加 `owner_id`（agent 归属）
- 权限系统（admin=`*`，可配置权限，统一中间件）
- 频道删除（作为权限系统的第一个应用）
- 加入 agent 的归属校验

### Phase 2：多用户注册与管理

- 用户注册流程（邀请制 or 开放注册）
- 用户管理自己的 agent（创建/配置/权限）
- 用户间 DM 和频道协作

### Phase 3：画布 + 文档协作（原始愿景）

- 无限画布 + Markdown 卡片
- 浮动对话锚定到文档
- AI 协作助手
