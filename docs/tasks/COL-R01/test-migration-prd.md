# COL-R01 集成测试迁移 — PRD

日期：2026-04-26 | 状态：Draft

## 背景

Go server 重写已完成核心功能开发，但 TS server 积累的集成测试（覆盖认证、频道管理、消息收发、WebSocket 实时推送、权限控制等关键业务场景）尚未迁移到 Go。在删除 TS server 代码并切换生产环境之前，必须确保 Go server 的测试覆盖不低于 TS server。

## 目标用户

- 开发团队：通过测试保障功能对等，安全切换到 Go server

## 核心需求

### 需求 1: TS 集成测试场景 1:1 迁移

- 用户故事：作为开发者，我需要 Go server 的测试覆盖 TS server 已有的所有集成测试场景，以确保 Go 重写的功能完整性
- 验收标准：
  - [ ] TS server 中每个集成测试场景在 Go server 中都有对应的测试
  - [ ] 所有 Go 测试 PASS（含 race 检测）
  - [ ] 覆盖率 ≥ 85%（CI 门禁已有）

### 需求 2: 场景清单梳理

- 用户故事：作为架构师，我需要一份 TS 测试场景 → Go 测试对照表，以追踪迁移进度和验收完整性
- 验收标准：
  - [ ] 对照表列出 TS 的每个测试场景及其在 Go 中的对应状态
  - [ ] 对照表随迁移进度持续更新

## 不在 v1 范围

- 新增 TS 没有的测试场景（只迁移已有的）
- 性能测试 / 压力测试
- 前端测试
- 测试框架选型讨论（沿用 Go 标准 testing 包）

## 验收标准

- [ ] TS 中每个集成测试场景在 Go 中都有对应测试
- [ ] 所有 Go 测试 PASS
- [ ] 覆盖率 ≥ 85%
- [ ] 场景对照表完整且无遗漏
