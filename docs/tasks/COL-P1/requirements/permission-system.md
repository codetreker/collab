# Collab 权限系统 — PRD

日期：2026-04-20 | 状态：In Review

## 背景

Collab 当前只有 admin/agent 两种角色，权限逻辑散落在各处代码中，没有统一的权限体系。随着频道删除等新功能的加入，需要一套系统化的权限模型来支撑当前和未来的权限需求。

## 设计原则

- **简洁** — 角色只区分"谁能管权限"，权限本身是独立可配置的能力
- **够用** — 覆盖当前已有功能和近期可预见的权限需求
- **可扩展** — admin = `*` 通配，未来新增权限 admin 自动拥有；member/agent 按需赋予

## 目标用户

| 角色 | 描述 |
|------|------|
| admin | 系统管理员，拥有全局管理权限 |
| member | 普通用户，日常使用者 |
| agent | AI bot，参与频道对话但不拥有管理权限 |

## 核心需求

### 需求 1: 角色与权限模型

**核心概念**：角色只决定"谁能管权限"，权限是独立的、可配置的能力。

**角色**（3 种）：
- **admin**：权限 = `*`（通配，自动拥有所有权限，包括未来新增的。本质是"权限的管理者"）
- **member**：可被 admin 赋予各种权限（"权限的使用者"）
- **agent**：和 member 一样可被赋予各种权限

**权限列表**（v1）：
- `channel.create` — 创建频道
- `channel.delete` — 删除频道
- `channel.manage_members` — 管理频道成员
- `channel.manage_visibility` — 修改频道可见性
- `message.send` — 发送消息
- `message.delete` — 删除消息

**admin 专属能力**（通过 `*` 通配获得，不需要单独配置）：
- `admin.manage_users` — 管理用户
- `admin.manage_permissions` — 管理权限

**默认权限**（新用户创建时自动赋予）：
- 新建 member：`channel.create` + `message.send`
- 新建 agent：`message.send`
- admin：`*`（不走默认权限列表）

**权限检查逻辑**：
```
if (user.role === 'admin') return true;  // * 通配
return user.permissions.includes(requiredPermission);
```

**用户故事**：
- 作为管理员，我想给某个 member 赋予"删除频道"的权限而不需要把他提升为 admin
- 作为管理员，我想给 agent 赋予"创建频道"的权限

**验收标准**：
- [ ] admin 自动拥有所有权限，无需逐条配置
- [ ] admin 可以给 member/agent 添加或移除权限
- [ ] 权限变更立即生效
- [ ] 管理页面有权限配置界面
- [ ] 新用户按角色获得默认权限集
- [ ] 至少保留一个 admin（不能把最后一个 admin 降级）

---

### 需求 2: 频道级权限

**概述**：权限不再和角色绑死，而是按用户实际拥有的权限判断。

**权限映射表**：

| 操作 | 需要的权限 | 额外条件 |
|------|-----------|----------|
| 创建频道 | `channel.create` | - |
| 删除频道 | `channel.delete` | #general 不可删，DM 不可删 |
| 修改可见性 | `channel.manage_visibility` | #general 不可改为私有 |
| 管理成员 | `channel.manage_members` | - |
| 发送消息 | `message.send` | 需是频道成员 |
| 查看消息 | （所有成员） | admin 可查看所有频道 |

**特殊规则**：
- `#general` 频道不可删除、不可改为私有
- DM 频道不可删除
- 私有频道：非成员返回 404（不泄漏存在），admin 除外（admin 可看所有频道）

**用户故事**：
- 作为拥有 `channel.delete` 权限的用户，我可以删除频道
- 作为没有 `channel.delete` 权限的用户，我看不到删除按钮

**验收标准**：
- [ ] 权限映射表中每个操作按用户实际权限正确判断
- [ ] 无权操作返回 403（私有频道非成员返回 404）
- [ ] 前端根据用户权限动态显示/隐藏操作按钮
- [ ] admin 无需配置任何权限即可执行所有操作

---

### 需求 3: 频道删除

**概述**：支持 admin 和频道创建者删除频道，删除后通知所有在线成员并自动跳回 #general。

**删除规则**：

- admin 可以删除任何频道（#general 除外）
- 频道创建者（member）可以删除自己创建的频道
- 普通成员和 agent 不能删除频道

**交互流程**：

1. 用户点击频道设置中的"删除频道"按钮
2. 弹出确认弹窗，显示频道名称，要求二次确认
3. 用户确认后，后端执行删除
4. 后端通过 WebSocket / SSE 事件通知所有该频道的在线成员
5. 客户端收到删除事件后，自动将用户导航到 #general

**用户故事**：

> 作为管理员，我想删除不再需要的频道，以便保持频道列表整洁。

> 作为频道创建者，我想在频道不再需要时删除它，以便不占用频道列表空间。

**验收标准**：

- [ ] admin 可以删除任何频道（#general 除外）
- [ ] 创建者可以删除自己创建的频道
- [ ] 普通成员和 agent 不能删除频道（按钮不显示或禁用）
- [ ] 删除前有确认弹窗，包含频道名称
- [ ] 删除后所有在线成员通过 WS/SSE 收到频道删除事件
- [ ] 收到删除事件的客户端自动跳回 #general
- [ ] 频道侧边栏实时移除已删除的频道

## 成功指标

- 所有权限检查通过统一的权限服务完成，不再散落在各处
- 权限矩阵中定义的每个操作行为正确且有测试覆盖
- 新增权限点只需在权限矩阵中添加一行，无需修改框架
