# B07: Agent 自定义 Slash Commands — PRD

日期：2026-04-24 | 状态：Draft

## 背景

P6 实现了 Slash Commands v1：5 个硬编码的内置命令（/help /invite /leave /topic /dm），架构上已预留 `CommandRegistry` 扩展点。

Collab 的核心定位是 Human-Agent 协作平台，Agent 是一等公民。Agent 需要一种标准化的方式向用户暴露自己的能力——不是靠用户记住"@agent 帮我搜xxx"，而是在输入框输入 `/` 就能发现 Agent 提供了什么操作、每个操作需要什么参数。

B07 实现 Agent 自定义 Slash Commands：Agent 通过 API 注册命令，用户在输入框通过 `/` 触发。

## 目标用户

- **Agent owner**：为自己的 Agent 注册自定义命令，让用户能方便地调用 Agent 能力
- **人类用户**：在聊天输入框输入 `/` 发现并使用 Agent 提供的命令
- **Agent 开发者**：通过 API 或 Plugin SDK 注册命令

## 核心需求

### 需求 1: Agent 命令注册

**用户故事**：作为 Agent owner，我想为我的 Agent 注册自定义命令（如 /deploy、/search、/ask），以便频道里的用户能快速触发 Agent 的能力。

**详细说明**：

- Agent 通过 API 注册命令，每个命令包含：
  - **命令名**（如 `search`，用户输入 `/search`）
  - **描述**（如"搜索知识库"，显示在命令列表里帮助用户理解）
  - **参数定义**（可选）：参数名、类型（text / user / number）、是否必填、描述
- 一个 Agent 可以注册多个命令
- 命令名全局唯一还是 Agent 级唯一？→ **Agent 级唯一**（不同 Agent 可以注册同名命令，显示时带 Agent 名前缀区分）
- Agent 可以更新或删除已注册的命令
- 命令注册后对 Agent 所在的所有频道生效

**验收标准**：

- [ ] Agent 通过 API 注册命令成功，返回命令 ID
- [ ] 注册时缺少必填字段（name / description）返回 400
- [ ] 同一 Agent 注册重复命令名返回冲突错误
- [ ] Agent 可以更新已注册命令的描述和参数
- [ ] Agent 可以删除已注册的命令
- [ ] 非 owner 不能修改/删除 Agent 的命令

### 需求 2: 命令发现（用户端）

**用户故事**：作为用户，我在输入框打 `/` 时，想看到当前频道所有可用命令（内置 + Agent 注册的），以便快速找到我需要的操作。

**详细说明**：

- 输入 `/` 弹出命令列表，包含：
  - 内置命令（P6 的 /help /invite 等）
  - **当前频道内所有 Agent 注册的命令**
- 每个 Agent 命令显示：
  - 命令名（如 `/search`）
  - 描述（如"搜索知识库"）
  - **来源 Agent 名**（如"by SearchBot"），帮助用户区分不同 Agent 的同名命令
- 列表分组：先内置命令，再按 Agent 分组
- 输入字符过滤命令（与 v1 行为一致）
- 命令列表实时更新：Agent 新注册/删除命令后，用户不需要刷新页面

**验收标准**：

- [ ] 输入 `/` 时列表包含内置命令和 Agent 命令
- [ ] Agent 命令显示来源 Agent 名
- [ ] 不同 Agent 的同名命令都能显示，用 Agent 名区分
- [ ] 只显示当前频道内 Agent 的命令（Agent 不在频道则命令不显示）
- [ ] Agent 新注册命令后，用户不刷新即可在列表中看到

### 需求 3: 命令执行

**用户故事**：作为用户，我选择一个 Agent 命令并输入参数后，想在聊天里看到 Agent 的执行结果。

**详细说明**：

- 用户选择命令后，如果命令有参数，显示参数输入提示（复用 v1 参数输入逻辑）
- 用户提交后，命令请求发送给对应的 Agent
- Agent 处理命令后，结果作为 Agent 的消息显示在聊天中
- 命令执行过程中，显示"Agent 正在处理…"状态指示
- Agent 未在线或处理超时时，显示错误提示
- 命令调用记录不作为普通消息持久化（仅结果消息持久化）

**验收标准**：

- [ ] 选择 Agent 命令 + 输入参数 + 提交 → Agent 收到命令请求
- [ ] Agent 处理后结果显示在聊天中
- [ ] 命令执行期间显示加载状态
- [ ] Agent 不在线时提示"Agent 离线，无法执行命令"
- [ ] 处理超时（30 秒无响应）时提示超时

### 需求 4: 命令参数

**用户故事**：作为用户，我想在使用带参数的 Agent 命令时，有清晰的参数提示，以便正确输入。

**详细说明**：

- 参数类型支持：
  - `text`：自由文本（如搜索关键词）
  - `user`：选择用户（复用 MentionPicker）
  - `number`：数字输入
- 必填参数未填时阻止提交，显示提示
- 参数 placeholder 来自命令注册时的描述

**验收标准**：

- [ ] text 参数显示 placeholder 提示
- [ ] user 参数弹出用户选择器
- [ ] 必填参数未填时无法提交，显示提示
- [ ] 多参数命令按顺序引导输入

## 不在范围

- 命令权限控制（哪些用户能用哪些命令）→ 后续版本
- 命令使用统计/分析
- 命令别名（alias）
- 命令分类/标签
- 复杂参数类型（文件上传、日期选择、下拉选项等）
- Agent 间命令互调
- 命令历史 / 最近使用
- 命令输出的富格式（仅支持 Markdown 文本，不支持自定义 UI 卡片）

## 成功指标

- Agent 命令注册数 > 5（说明 Agent 开发者在用）
- 用户通过 `/` 命令触发 Agent 的频率 > 直接 @ Agent 频率的 20%
- 命令执行成功率 > 95%

## 开放问题

1. **命令名冲突策略**：如果内置命令和 Agent 命令同名怎么办？→ 建议内置命令优先，Agent 命令需要用 `/agent-name:command` 格式调用（待讨论）
2. **命令数量上限**：单个 Agent 最多注册多少命令？→ 建议 v1 限制 20 个
3. **Agent 命令批量注册**：Agent 连接时一次性注册所有命令，还是按需逐个注册？→ 待技术设计确定
