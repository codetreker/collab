# COL-B10: 消息编辑与删除 — PRD

日期：2026-04-21 | 状态：Approved

## 背景
用户发消息后无法修改或删除，发错了只能发新消息更正。这是聊天平台的基本功能。

## 功能 1: 消息编辑
**用户故事**：作为用户，我想编辑已发送的消息来修正错误。
**需求**：
- 鼠标悬浮自己的消息时显示"编辑"按钮（或右键菜单）
- 点击后消息变为可编辑状态（inline 编辑，不弹窗）
- Enter 保存，Esc 取消
- 编辑后消息显示"(已编辑)"标记 + 编辑时间
- 只能编辑自己的消息
- Admin 不能编辑别人的消息（只能删除）
- 编辑通过 WS 实时同步给其他用户
- API：PUT /api/v1/messages/:id
- DB：messages 表加 edited_at 字段
**验收标准**：
- [ ] 鼠标悬浮显示编辑按钮
- [ ] inline 编辑，Enter 保存，Esc 取消
- [ ] 编辑后显示"(已编辑)"
- [ ] WS 实时同步
- [ ] 只能编辑自己的消息

## 功能 2: 消息删除
**用户故事**：作为用户，我想删除已发送的消息。
**需求**：
- 鼠标悬浮自己的消息时显示"删除"按钮
- 点击后确认弹窗"确定删除这条消息？"
- 软删除（设 deleted_at，不硬删）
- 删除后其他用户看到"此消息已删除"（灰色斜体）
- 只能删除自己的消息
- Admin 可以删除任何人的消息（频道管理需要）
- 删除通过 WS 实时同步
- API：DELETE /api/v1/messages/:id
- DB：messages 表加 deleted_at 字段
**验收标准**：
- [ ] 鼠标悬浮显示删除按钮
- [ ] 确认弹窗
- [ ] 删除后显示"此消息已删除"
- [ ] WS 实时同步
- [ ] 只能删除自己的消息
- [ ] Admin 可删除任何消息

## 消息操作栏
- 鼠标悬浮消息时右上角显示操作栏（和 reactions 按钮同一行）
- 操作项：reaction 😊 | 编辑 ✏️ | 删除 🗑️
- 编辑和删除只在自己的消息上显示
- Admin 在所有消息上显示删除

## Agent 行为
- Agent 可通过 API 编辑/删除自己的消息（和人类用户相同权限）
- 编辑/删除发生时，往频道消息流注入系统消息（type = `system`）：
  - 编辑：“用户 A 编辑了消息：[新内容]”
  - 删除：“用户 A 删除了一条消息”
- Agent 通过 SSE/poll 拿消息时自然获取这些变更，不需要额外事件机制
- v1 不强制 Agent 对编辑事件做出反应，只是通知

## 不在范围
- 编辑历史查看（v2）
- 批量删除
- 消息撤回时间限制（v1 不限时间）
- 频道级别的消息清理

## 成功指标
- 编辑/删除功能使用率 > 10%
