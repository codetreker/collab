# COL-B26 / B27: 频道拖动排序 + 自定义分组 — PRD

日期：2026-04-25 | 状态：Draft | 版本：v4

## 背景

Collab 的左侧 sidebar 频道列表目前按最近活跃时间排序，不支持手动排序和分组。随着频道增多，频道 owner 无法按项目/话题组织自己的频道，也无法把最常用的频道固定在顶部。

建军要求：
1. 频道可以拖动调整顺序（B26）
2. 频道可以自定义分组，类似 Discord 的 Category（B27）

## 核心概念

> **排序和分组是频道的属性，不是个人偏好。**
>
> - 每个频道有一个创建者（owner）
> - 只有频道 owner 能调整自己频道的排序位置和分组归属
> - 排序和分组的结果对所有用户一致——所有人看到的频道顺序相同
> - 其他用户只读，不能拖动别人的频道

## 目标用户

- **频道 owner（创建者）**：管理自己创建的频道的排序位置和分组归属
- **普通用户**：按 owner 设定的顺序查看频道列表（只读）

---

## 需求 1：频道拖动排序（B26）

### 用户故事

> 作为频道 owner，我想拖动调整我的频道在 sidebar 的显示顺序，以便把最重要的频道放在最容易看到的位置。

### 功能描述

| 项 | 说明 |
|---|---|
| 谁能排序 | **频道 owner（创建者）**。只能拖动自己创建的频道 |
| 排序范围 | 全局排序——owner 调整后，所有用户看到的顺序同步更新 |
| 拖动交互 | 桌面端：鼠标按住拖动手柄（≡）拖动；移动端：长按 ≥300ms 触发拖动 |
| 拖动手柄 | **仅在当前用户是 owner 的频道上显示**，非 owner 的频道不显示手柄 |
| 视觉反馈 | 拖影（半透明的频道项跟随光标/手指）+ 插入位置指示器（蓝色细线） |
| 持久化 | 排序结果服务端持久化（全局，非 per-user） |
| 与分组关系 | 频道可在分组之间拖动（B27），拖入分组即同时更新分组归属和排序位置 |
| 新建频道 | 默认排在所属分组末尾 |

### 验收标准

- [ ] 频道 owner 可拖动自己的频道，松手后顺序即时生效
- [ ] 非 owner 用户看不到拖动手柄，无法拖动别人的频道
- [ ] 拖动过程有拖影和插入位置蓝线指示
- [ ] 刷新页面后排序不变（服务端持久化）
- [ ] 排序变更对所有在线用户即时可见
- [ ] 移动端长按 ≥300ms 可触发拖动（仅 owner 的频道）
- [ ] 新建频道自动排在所属分组末尾

## 需求 2：频道自定义分组（B27）

### 用户故事

> 作为频道 owner，我想创建分组（Category）并把我的频道归入不同分组，以便按项目或话题组织频道。

> 作为用户，我想折叠/展开分组，以便隐藏不关注的频道、减少视觉噪音。

### 功能描述

| 项 | 说明 |
|---|---|
| 分组概念 | 类似 Discord 的 Category：一个有名字的容器，包含若干频道 |
| 谁能管理 | **频道 owner** 可创建/重命名/删除分组，并将自己的频道拖入分组 |
| 分组排序 | 分组之间可拖动排序（同 B26 拖动交互），仅分组创建者可拖动 |
| 频道归属 | 每个频道可属于一个分组，也可不属于任何分组 |
| 谁的频道入谁的组 | Owner 只能把自己的频道放入分组；不能操作别人的频道 |
| 未分组频道 | 显示在所有分组上方的 "未分组" 区域（无 header） |
| 折叠/展开 | 所有用户均可折叠/展开分组，折叠状态存客户端本地（localStorage） |
| 创建分组 | Owner 在 sidebar 点 "+" → 弹窗输入分组名 → 创建 |
| 重命名 | 右键分组 header → "重命名" → inline 编辑（仅分组创建者可见） |
| 删除分组 | 右键分组 header → "删除分组" → 确认弹窗 → 组内频道回到 "未分组"（仅分组创建者可见） |
| 嵌套 | **不支持**——分组不能嵌套分组 |

### 验收标准

- [ ] 频道 owner 可创建分组，输入名称后 sidebar 即时出现新分组 header
- [ ] 频道 owner 可将自己的频道拖入/拖出分组
- [ ] 非 owner 用户不能创建分组、不能拖动频道、不能管理分组（只读视图）
- [ ] 分组 header 显示名称 + 折叠箭头（▾ / ▸）
- [ ] 所有用户可折叠/展开分组，刷新后折叠状态保持（localStorage）
- [ ] 分组创建者右键分组 header 可重命名、删除；非创建者无此菜单
- [ ] 删除分组后，原组内频道自动回到 "未分组" 区域
- [ ] 未分组频道显示在所有分组上方
- [ ] 分组不能嵌套分组
- [ ] 排序/分组变更后，所有在线用户实时看到更新后的顺序

---

## 不在 v1 范围

- 频道嵌套（分组不能再嵌套分组）
- 自动分组规则（按标签/类型自动归组）
- DM 列表排序/分组
- 频道置顶/收藏功能（可作为后续需求）
- 分组模板/预设（提供默认分组方案）
- 个人偏好排序（per-user 排序视图）

## 成功指标

| 指标 | 目标 |
|---|---|
| 频道 owner 能在 10 秒内完成一次频道拖动排序 | ≤ 10s |
| 频道 owner 能在 30 秒内创建分组并拖入 3 个频道 | ≤ 30s |
| 排序/分组操作保存后刷新页面状态不变 | 100% |
| 排序变更后其他用户 sidebar 在 2 秒内更新 | ≤ 2s |
| 折叠/展开操作无感知延迟 | ≤ 100ms |

## 开放问题

1. ~~排序全局还是每人独立？~~ → **全局**。排序是频道属性，owner 设定，所有人看到一样的顺序
2. ~~是否需要拖动权限细分？~~ → **按 owner 区分**。只有频道创建者能拖动/分组，其他人只读
3. 分组数量上限？→ v1 不限制，后续根据使用情况决定
4. 新频道默认排序？→ 未手动排序前按最近活跃时间降序（与现有行为一致）
5. 多个 owner 的频道混排时排序冲突？→ 每个 owner 只能移动自己的频道，排序在全局空间内编排，不同 owner 的频道可交错排列
