# COL-B26 / B27: 频道拖动排序 + 自定义分组 — PRD

日期：2026-04-25 | 状态：Draft

## 背景

Collab 的左侧 sidebar 频道列表目前按最近活跃时间排序，不支持手动排序和分组。随着频道增多，用户无法按项目/话题组织频道，也无法把最常用的频道固定在顶部。

建军要求：
1. 频道可以拖动调整顺序（B26）
2. 频道可以自定义分组，类似 Discord 的 Category（B27）

## 目标用户

- **所有用户**：管理自己的频道排序、创建/编辑/删除自己的分组、调整分组内频道顺序、折叠/展开分组

> 排序和分组都是**个人偏好**，每个用户独立管理自己的视图，互不影响。

## 现状

参见 `packages/client/src/components/Sidebar.tsx`：

- 频道列表 `sortedChannels` 按 `last_message_at` 降序排列
- 已加入频道平铺显示，未加入公开频道在 "公开频道" 分隔线下方
- DM 列表独立区域
- 无任何排序/分组概念，服务端无 `position` 或 `group` 字段

---

## 需求 1：频道拖动排序（B26）

### 用户故事

> 作为用户，我想拖动频道调整它在 sidebar 的显示顺序，以便把最重要的频道放在最容易看到的位置。

### 功能描述

| 项 | 说明 |
|---|---|
| 谁能排序 | **所有用户**。每个用户独立管理自己的频道排序 |
| 排序范围 | 个人排序（每人独立），不影响其他用户 |
| 拖动交互 | 桌面端：鼠标按住拖动；移动端：长按 ≥300ms 触发拖动 |
| 视觉反馈 | 拖影（半透明的频道项跟随光标/手指）+ 插入位置指示器（蓝色细线） |
| 持久化 | 排序结果存服务端，`user_channel_preferences` 表记录每用户的 `position` |
| 与分组关系 | 频道可在分组之间拖动（B27），拖入分组 = 设置 `group_id` + 更新 `position` |
| 新建频道 | 默认 `position` 为最大值 +1（排在末尾） |

### 验收标准

- [ ] 任何用户在 sidebar 可拖动频道，松手后顺序即时生效
- [ ] 拖动过程有拖影和插入位置蓝线指示
- [ ] 刷新页面后排序不变（服务端持久化）
- [ ] 排序是个人偏好，不影响其他用户的频道顺序
- [ ] 移动端长按 ≥300ms 可触发拖动
- [ ] 新建频道自动排在所属分组末尾

---

## 需求 2：频道自定义分组（B27）

### 用户故事

> 作为用户，我想创建分组（Category）并把频道归入不同分组，以便按项目或话题组织频道。

> 作为用户，我想折叠/展开分组，以便隐藏不关注的频道、减少视觉噪音。

### 功能描述

| 项 | 说明 |
|---|---|
| 分组概念 | 类似 Discord 的 Category：一个有名字的容器，包含若干频道 |
| 谁能管理 | **所有用户**均可创建/重命名/删除自己的分组（个人偏好） |
| 分组排序 | 分组之间可拖动排序（同 B26 拖动交互） |
| 频道归属 | 每个频道可属于一个分组，也可不属于任何分组 |
| 未分组频道 | 显示在所有分组上方的 "未分组" 区域（无 header） |
| 折叠/展开 | 所有用户均可折叠/展开分组，折叠状态存客户端本地（localStorage） |
| 创建分组 | 用户在 sidebar 点 "+" → 弹窗输入分组名 → 创建 |
| 重命名 | 右键分组 header → "重命名" → inline 编辑 |
| 删除分组 | 右键分组 header → "删除分组" → 确认弹窗 → 组内频道回到 "未分组" |
| 嵌套 | **不支持**——分组不能嵌套分组 |

### 数据模型（建议）

排序和分组都是个人偏好，存到用户维度的表中，不修改全局 `channels` 表。

```
user_channel_preferences 表:
  user_id     TEXT NOT NULL
  channel_id  TEXT NOT NULL
  group_id    TEXT REFERENCES user_channel_groups(id) ON DELETE SET NULL
  position    INTEGER NOT NULL DEFAULT 0
  PRIMARY KEY (user_id, channel_id)

user_channel_groups 表:
  id          TEXT PRIMARY KEY
  user_id     TEXT NOT NULL
  name        TEXT NOT NULL
  position    INTEGER NOT NULL DEFAULT 0
  created_at  INTEGER NOT NULL
  updated_at  INTEGER NOT NULL
```

> 每个用户有自己独立的分组和排序数据，互不影响。未设置过偏好的用户看到默认排序（按 `last_message_at` 降序）。

### API（建议）

> 接口不在 `/admin/` 路径下——排序和分组是所有用户的功能。操作自动关联当前登录用户。

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/channel-groups` | 创建分组（当前用户） |
| PATCH | `/api/v1/channel-groups/:id` | 重命名分组（当前用户） |
| DELETE | `/api/v1/channel-groups/:id` | 删除分组（频道回到未分组） |
| PUT | `/api/v1/channel-groups/reorder` | 批量更新分组排序 |
| PUT | `/api/v1/channels/reorder` | 批量更新频道排序 + 分组归属 |

### WS 事件

> 排序和分组是个人偏好，不需要实时广播给其他用户。以下事件仅用于同一用户的多设备同步。

| 事件 | Payload | 说明 |
|---|---|---|
| `user_channel_group_changed` | `{ groups, preferences }` | 用户自己的分组/排序变更，同步到该用户的其他在线设备 |

### 验收标准

- [ ] 任何用户可创建分组，输入名称后 sidebar 即时出现新分组 header
- [ ] 任何用户可将频道拖入/拖出自己的分组
- [ ] 分组 header 显示名称 + 折叠箭头（▾ / ▸）
- [ ] 所有用户可折叠/展开分组，刷新后折叠状态保持
- [ ] 右键分组 header 可重命名、删除
- [ ] 删除分组后，原组内频道自动回到 "未分组" 区域
- [ ] 未分组频道显示在所有分组上方
- [ ] 分组不能嵌套分组
- [ ] 分组是个人偏好，每个用户的分组独立，不影响其他用户

---

## 不在 v1 范围

- 频道嵌套（分组不能再嵌套分组）
- 自动分组规则（按标签/类型自动归组）
- DM 列表排序/分组
- 频道置顶/收藏功能（可作为后续需求）
- 分组模板/预设（提供默认分组方案）

## 成功指标

| 指标 | 目标 |
|---|---|
| 用户能在 10 秒内完成一次频道拖动排序 | ≤ 10s |
| 用户能在 30 秒内创建分组并拖入 3 个频道 | ≤ 30s |
| 排序/分组操作保存后刷新页面状态不变 | 100% |
| 折叠/展开操作无感知延迟 | ≤ 100ms |
