# COL-B03: 公开频道预览 + 加入 — 任务分解

日期：2026-04-21 | 状态：Ready

## 任务列表

### T1: 后端 — 频道列表返回未加入的公开频道

**改动文件**：`queries.ts`、`routes/channels.ts`

**内容**：修改 getChannelsForUser，UNION 未加入的公开频道

**验收标准**：
- [ ] API 返回已加入 + 未加入的公开频道
- [ ] is_member 正确标记
- [ ] 私有频道不返回给非成员

---

### T2: 后端 — 频道预览 API

**改动文件**：`routes/channels.ts`、`queries.ts`

**内容**：GET /channels/:id/preview，返回 24h 消息

**验收标准**：
- [ ] 公开频道可预览
- [ ] 私有频道返回 404
- [ ] 最多返回 50 条 24h 内消息

---

### T3: 后端 — 用户自加入公开频道（agent 限制）

**改动文件**：`routes/channels.ts`

**内容**：
1. 确认 POST members API 允许用户自己加入公开频道
2. **[HIGH]** Agent 不可自行加入公开频道——后端对 `user.role === 'agent'` 的自加入请求返回 403，agent 加入公开频道必须由频道 owner 操作（POST /channels/:id/members 由 owner 传入 agent 的 userId）

**验收标准**：
- [ ] 普通用户可以把自己加入公开频道
- [ ] 私有频道自加入被拒
- [ ] Agent 自加入公开频道返回 403
- [ ] Owner 可以将 agent 加入公开频道

---

### T4: 前端 — 侧边栏显示未加入频道（预览入口）

**改动文件**：侧边栏组件、`index.css`

**内容**：
1. 未加入公开频道灰色显示 + "预览"标签
2. **[HIGH]** 移除侧边栏中未加入频道上现有的"加入"按钮——点击未加入频道直接进入预览模式，用户在预览模式内通过底部"加入频道"按钮加入（见 T5）

**验收标准**：
- [ ] 未加入频道可见且样式区分
- [ ] 已加入和未加入分组
- [ ] 侧边栏未加入频道无"加入"按钮，点击整行进入预览模式

---

### T5: 前端 — 预览模式 + 加入按钮 + 加入后加载完整历史

**改动文件**：`ChannelView.tsx`、`MessageInput.tsx`、`AppContext.tsx`

**内容**：
1. 非成员点击频道 → 加载预览 API（24h 消息只读）
2. 只读消息列表 + 底部"加入频道"按钮（替换输入框）
3. **[HIGH]** 加入成功后：替换预览消息，调用正常的消息历史 API 加载完整消息（而非继续显示 24h 预览数据），然后 WS subscribe

**验收标准**：
- [ ] 预览模式显示 24h 消息
- [ ] 无法在预览模式发消息
- [ ] 加入后加载完整消息历史（替换 24h 预览数据）
- [ ] 加入后可正常使用
