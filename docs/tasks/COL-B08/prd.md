# B08: 内置系统命令（/status /clear /nick）— PRD

日期：2026-04-24 | 状态：Draft

## 背景

P6 实现了 5 个内置 Slash Commands：/help /invite /leave /topic /dm。其中 /status、/clear、/nick 作为 v2 预留，未实现。

这三个命令是常用的系统级操作——查看频道状态、清理本地聊天历史、修改显示名。它们不需要 Agent 参与，由客户端或服务端直接处理。

## 目标用户

- **人类用户**：快速执行系统操作，不需要找设置页面或菜单

## 核心需求

### 需求 1: /status — 显示频道状态

**用户故事**：作为用户，我想快速查看当前频道的状态信息（谁在线、有哪些 Agent、频道基本信息），以便了解当前协作环境。

**详细说明**：

- 输入 `/status` 后，在聊天区显示一条本地系统消息，内容包含：
  - **频道信息**：频道名、频道类型（公开/私有）、频道主题（topic，如有）、创建时间
  - **成员概览**：总成员数、当前在线人数
  - **在线成员列表**：显示当前在线的用户和 Agent（区分标注）
  - **Agent 列表**：频道内所有 Agent 及其在线状态
- 这是本地消息，不持久化到服务端，其他用户看不到
- 不带参数，直接执行

**验收标准**：

- [ ] 输入 `/status` 后在聊天区看到频道状态信息
- [ ] 状态信息包含频道名、类型、成员数、在线人数
- [ ] 在线用户和 Agent 分别列出
- [ ] Agent 有明确标识（如 🤖 图标）
- [ ] 信息为本地消息，刷新页面后消失
- [ ] DM 频道中也能使用（显示对方在线状态）

### 需求 2: /clear — 清除本地聊天记录

**用户故事**：作为用户，我想清除当前频道的本地聊天记录来获得一个干净的聊天界面，但不影响服务端存储的消息。

**详细说明**：

- 输入 `/clear` 后：
  1. 弹出确认对话框："确定清除本频道的本地聊天记录？消息不会从服务器删除，刷新页面后会重新加载。"
  2. 用户确认后，清除当前频道在本地内存中的所有消息
  3. 聊天区变空，显示一条系统提示："聊天记录已清除。刷新页面可恢复。"
- **只清除本地**：不发送任何删除请求到服务端
- **只清除当前频道**：不影响其他频道的消息
- 取消确认则不执行任何操作

**验收标准**：

- [ ] 输入 `/clear` 后弹出确认对话框
- [ ] 确认后当前频道聊天区清空
- [ ] 清空后显示提示信息
- [ ] 刷新页面后消息重新加载（验证服务端数据未删）
- [ ] 其他频道的消息不受影响
- [ ] 点击取消不清除任何内容

### 需求 3: /nick — 修改显示名

**用户故事**：作为用户，我想快速修改自己的显示名，不用去找设置页面。

**详细说明**：

- 输入 `/nick <新名字>` 后，修改当前用户的显示名（displayName）
- 修改后：
  - 本地立即生效（头部、侧边栏、消息中的名字更新）
  - 通过 API 持久化到服务端
  - 通过 WebSocket 通知其他在线用户，实时更新该用户在所有频道的显示名
- 显示名限制：
  - 长度：2-32 字符
  - 不能为空或纯空格
  - 不做敏感词过滤（v1）
- 修改成功后显示本地系统消息："显示名已修改为 XXX"
- 参数缺失时提示："用法：/nick <新名字>"

**验收标准**：

- [ ] `/nick 新名字` 修改成功，本地立即显示新名字
- [ ] 修改持久化到服务端（刷新后仍是新名字）
- [ ] 其他在线用户实时看到名字变化
- [ ] 名字长度 < 2 或 > 32 时提示错误
- [ ] 不输入名字时提示用法
- [ ] 修改成功后显示确认系统消息

## 不在范围

- /status 显示用户活跃度/最后在线时间
- /clear 支持参数（如清除最近 N 条、清除某时间段）
- /clear 清除服务端消息
- /nick 头像修改
- /nick 敏感词过滤
- /nick 频道级昵称（不同频道不同名字）
- 个人状态文本设置（如"正在开会"，这是另一个功能）

## 与 v1 命令列表的关系

B08 完成后，内置命令列表变为 8 个：

| 命令 | 来源 | 参数 | 说明 |
|------|------|------|------|
| /help | P6 v1 | 无 | 显示所有可用命令 |
| /invite | P6 v1 | @user | 邀请用户到当前频道 |
| /leave | P6 v1 | 无 | 离开当前频道 |
| /topic | P6 v1 | text | 设置频道主题 |
| /dm | P6 v1 | @user | 打开与某人的 DM |
| **/status** | **B08** | **无** | **显示频道状态** |
| **/clear** | **B08** | **无** | **清除本地聊天记录** |
| **/nick** | **B08** | **text** | **修改显示名** |

## 成功指标

- 命令使用率：/status + /clear + /nick 占所有命令使用的 > 15%
- /nick 替代设置页面修改名字的比例 > 50%

## 开放问题

1. **/clear 是否要支持参数？** 如 `/clear 50` 只清最近 50 条。→ 建议 v1 不支持，保持简单。用户需要时再迭代。
2. **/nick 是否需要通知频道？** 如"XXX 将名字改为 YYY"这种系统消息。→ 建议 v1 不做，避免频道刷屏。
