# COL-B22: 消息路径文件链接 — PRD

日期：2026-04-22 | 状态：Draft | 依赖：B21（Plugin WS）

## 背景
Agent 发消息时经常提到本地文件路径（如 `/workspace/collab/src/server.ts`）。目前这些路径是纯文本，用户无法直接查看文件内容。需要自动将路径转为可点击链接，点击后通过 Agent 的 Plugin WS 连接读取文件并在 FileViewer 中预览。

## 目标用户
- Agent 的 owner（只有 owner 能看到链接并读取文件）

## 核心需求

### 需求 1: 路径检测
- 用户故事：作为用户，我看到 Agent 消息中的文件路径时，它们是可点击的链接
- 验收标准：
  - [ ] 正则匹配绝对路径（`/xxx/yyy.ext`）
  - [ ] Agent 消息中的路径自动渲染为链接
  - [ ] 非 Agent 消息不做路径检测
  - [ ] 只对该 Agent 的 owner 显示为链接，其他人看到纯文本

### 需求 2: 通过 Plugin WS 读取文件
- 用户故事：作为 owner，我点击路径链接后能看到文件内容
- 验收标准：
  - [ ] 点击链接 → server 通过 WS 向 Agent 的 Plugin 发文件读取请求
  - [ ] Plugin 返回文件内容
  - [ ] FileViewer 渲染（复用 B20 的共享组件）
  - [ ] 读取超时显示提示

### 需求 3: Agent 离线处理
- 用户故事：作为用户，Agent 离线时我点击链接能看到合理提示
- 验收标准：
  - [ ] Agent 离线 → 提示"Agent 离线，无法读取文件"
  - [ ] 链接仍然显示但标记为不可用状态

### 需求 4: Plugin 端文件读取能力
- 用户故事：作为 Plugin 开发者，Plugin 能接收并响应文件读取请求
- 验收标准：
  - [ ] Plugin 收到文件读取请求，返回文件内容
  - [ ] 目录白名单控制（`~/.config/collab/` 配置允许访问的目录）
  - [ ] 白名单外的路径拒绝读取

## 不在范围
- 文件写入/修改（只读）
- 目录浏览（B19 Remote Explorer scope）
- 非 Agent 消息的路径检测

## 成功指标
- Agent 消息中的文件路径可点击率 > 90%
- 文件读取成功率（Agent 在线时）> 95%
