# COL-B20: Channel Workspace — PRD

日期：2026-04-22 | 状态：Draft

## 背景
Collab 目前没有文件管理功能。聊天中分享的文件散落在消息里，找不到。需要一个频道级的文件存储空间（Workspace），让用户上传、管理、预览文件。

## 目标用户
- Collab 所有用户（人类和 Agent）
- 频道成员可以共享文件

## 核心需求

### 需求 1: 文件上传与存储
- 用户故事：作为频道成员，我想把文件上传到频道 Workspace，以便团队成员都能访问
- 验收标准：
  - [ ] 支持任意文件类型上传
  - [ ] 单文件大小限制 10MB
  - [ ] 存储路径：`/data/workspaces/{userId}/{channelId}/`
  - [ ] 元数据存 SQLite（文件名、大小、上传者、时间）
  - [ ] 同名文件自动加后缀 `file (1).md`（除非明确 replace）

### 需求 2: 文件夹管理
- 用户故事：作为用户，我想创建文件夹组织文件
- 验收标准：
  - [ ] 支持创建/删除/重命名文件夹
  - [ ] 支持嵌套文件夹
  - [ ] 文件可移动到不同文件夹

### 需求 3: 文件预览
- 用户故事：作为用户，我想在浏览器里直接预览文件内容
- 验收标准：
  - [ ] Markdown → 渲染预览
  - [ ] 代码文件 → Shiki 语法高亮
  - [ ] 图片 → 查看器
  - [ ] 文本检测 → 纯文本 fallback
  - [ ] 二进制 → 提示"不支持预览"
  - [ ] 共享 FileViewer 组件（和 B19/B22 复用）

### 需求 4: 消息附件自动存入
- 用户故事：作为用户，我发消息附带的文件自动存入频道 Workspace
- 验收标准：
  - [ ] 聊天中发送的附件自动存入当前频道 Workspace
  - [ ] 在 Workspace 中可看到来源消息的关联

### 需求 5: Markdown 在线编辑
- 用户故事：作为用户，我想直接在浏览器编辑 Markdown 文件
- 验收标准：
  - [ ] `.md` 文件支持编辑模式
  - [ ] 保存后更新文件内容

### 需求 6: 聊天引用 Workspace 文件
- 用户故事：作为用户，我想在聊天中引用/嵌入 Workspace 里的文件
- 验收标准：
  - [ ] 支持在消息中引用 Workspace 文件（生成预览卡片）

### 需求 7: 统一管理页面
- 用户故事：作为用户，我想在一个页面看到我所有频道的 Workspace 文件
- 验收标准：
  - [ ] 统一管理页面，按频道分组
  - [ ] 按 user 隔离（只看自己有权限的频道）

### 需求 8: 前端侧边栏文件树
- 用户故事：作为用户，我想在聊天界面侧边看到文件树
- 验收标准：
  - [ ] 侧边栏文件树 UI
  - [ ] Remote / Workspace 两个 tab（和 B19 统一）

## 不在范围
- 文件版本控制 — v2
- 实时协同编辑（多人同时编辑）— v2
- 文件搜索（全文搜索）— v2
- 文件权限细粒度控制（v1 跟随频道权限）

## 成功指标
- 用户在 Workspace 上传文件 > 0
- 聊天附件自动存入率 100%
