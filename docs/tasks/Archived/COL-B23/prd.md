# COL-B23: 聊天记录分页加载 — PRD

日期：2026-04-22 | 状态：Draft

## 背景
当前进入频道时加载全部聊天记录，消息多了页面会卡顿。需要分页加载——初始只加载最近 100 条，往上滚动时增量加载更早的消息。

## 目标用户
所有 Collab 用户

## 核心需求

### 需求 1: 初始加载限制
- 用户故事：作为用户，我进入频道时页面快速加载，不卡顿
- 验收标准：
  - [ ] 进入频道只加载最近 100 条消息
  - [ ] 加载完成后自动滚动到最新消息
  - [ ] 加载速度明显快于加载全部

### 需求 2: 向上滚动增量加载
- 用户故事：作为用户，我想看更早的消息时，往上滚就能加载
- 验收标准：
  - [ ] 滚动到顶部时自动触发加载更早的消息
  - [ ] 每次加载 50 条（或可配置）
  - [ ] 加载时显示 loading 指示器
  - [ ] 加载完成后保持当前阅读位置（不跳到顶部）
  - [ ] 没有更早消息时显示"已到最早消息"提示

### 需求 3: 新消息处理
- 用户故事：作为用户，有人发新消息时我能看到
- 验收标准：
  - [ ] 新消息通过 WS/SSE 实时追加到底部
  - [ ] 如果用户在看历史消息（滚动位置不在底部），显示"新消息"浮动按钮
  - [ ] 点击"新消息"按钮滚动到最新

### 需求 4: API 分页
- 用户故事：作为 API 调用者，消息接口支持分页参数
- 验收标准：
  - [ ] GET /messages 支持 `limit` + `before`（cursor 分页）
  - [ ] 默认 limit=100
  - [ ] 返回 `hasMore` 标识是否还有更早消息

## 不在范围
- 消息搜索跳转到特定位置 — v2
- 未读消息跳转 — v2（和已读回执 B09 一起做）

## 成功指标
- 频道加载时间 < 1s（100 条以内）
- 滚动加载无卡顿
