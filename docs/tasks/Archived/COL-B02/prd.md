# COL-B02: 频道删除 UX 完善 — PRD

日期：2026-04-21 | 状态：Draft

## 背景

P1 已实现频道软删除后端（API `DELETE /channels/:id` + WS broadcast `channel_deleted`），但前端没有入口。用户无法通过 UI 删除频道。需要补齐前端交互。

## 目标用户

- 频道创建者 / 拥有 `channel.delete` 权限的成员
- 工作区 Admin

## 核心需求

### 需求 1: 删除按钮入口

- **用户故事**：作为有删除权限的用户，我想在频道设置中看到"删除频道"按钮，以便管理不再需要的频道。
- **详细描述**：
  - 在频道设置页面（Channel Settings）底部「危险区域」显示红色"删除频道"按钮
  - 仅当当前用户拥有 `channel.delete` 权限时才渲染该按钮
  - Admin 角色对所有频道均显示删除按钮
- **验收标准**：
  - [ ] 有 `channel.delete` 权限的用户在频道设置中看到删除按钮
  - [ ] 无权限用户看不到删除按钮
  - [ ] Admin 对任意频道均可见删除按钮

### 需求 2: 确认对话框

- **用户故事**：作为用户，我在点击删除后需要二次确认，避免误操作。
- **详细描述**：
  - 点击"删除频道"后弹出模态确认框
  - 标题："删除频道"
  - 正文："确定删除 **#频道名**？此操作不可恢复。"
  - 按钮：「取消」（secondary）+ 「删除」（danger / 红色）
  - 「删除」按钮默认不聚焦，防止回车误触
- **验收标准**：
  - [ ] 确认弹窗正确显示频道名称
  - [ ] 点击取消关闭弹窗，无副作用
  - [ ] 点击删除后发起 API 请求

### 需求 3: 调用 DELETE API

- **详细描述**：
  - 确认后调用 `DELETE /channels/:id`
  - 请求期间删除按钮显示 loading 状态，禁止重复点击
  - 失败时 Toast 提示错误信息，弹窗保持打开
- **验收标准**：
  - [ ] 成功调用 DELETE API
  - [ ] loading 期间按钮不可重复点击
  - [ ] API 失败显示错误 Toast

### 需求 4: 删除成功后的跳转与通知

- **用户故事**：作为用户，频道被删后我需要被安全引导到有效页面，不能停留在已删频道。
- **详细描述**：
  - **当前用户（执行删除者）**：删除成功后自动跳转到 `#general`，Toast 提示"频道已删除"
  - **其他在线成员**：收到 `channel_deleted` WS 事件后：
    - 若当前正在被删频道 → 自动跳转到 `#general` + Toast 提示"#频道名 已被删除"
    - 若不在被删频道 → 侧边栏移除该频道，不打断当前操作
  - 侧边栏频道列表实时移除被删频道
- **验收标准**：
  - [ ] 删除者跳回 `#general`
  - [ ] 其他在该频道的成员自动跳回 `#general` + Toast
  - [ ] 不在该频道的成员侧边栏静默移除
  - [ ] 频道列表实时更新

### 需求 5: 不可删除的频道

- **详细描述**：
  - `#general`（默认频道）：不显示删除按钮，无论权限
  - DM（私信）频道：不显示删除按钮
  - 判断逻辑：`channel.isDefault === true` 或 `channel.type === 'dm'` 时隐藏删除入口
- **验收标准**：
  - [ ] `#general` 设置页无删除按钮
  - [ ] DM 频道无删除按钮

### 需求 6: Admin 权限

- **详细描述**：
  - Admin 角色可以删除除 `#general` 外的任何频道
  - 权限判断优先级：`isDefault / isDM` 检查 > Admin 检查 > `channel.delete` 权限检查
- **验收标准**：
  - [ ] Admin 可删除任意非默认频道
  - [ ] Admin 也不能删除 `#general`

## 交互流程

```
用户打开频道设置
  → 有权限？显示"删除频道"按钮（红色，底部危险区域）
  → 点击"删除频道"
    → 弹出确认框："确定删除 #频道名？此操作不可恢复。"
    → 点击「取消」→ 关闭弹窗
    → 点击「删除」→ loading → 调用 DELETE API
      → 成功：关闭弹窗 → 跳转 #general → Toast "频道已删除"
      → 失败：Toast 错误提示，弹窗保持
```

## 验收标准汇总

- [ ] 有权限的用户在频道设置中看到删除按钮
- [ ] 确认弹窗正确显示频道名称
- [ ] 删除后当前用户跳回 `#general`
- [ ] 其他在该频道的成员收到通知 + 自动跳转
- [ ] `#general` 和 DM 无删除按钮
- [ ] Admin 可删除任何非默认频道
- [ ] API 请求期间有 loading 状态
- [ ] API 失败有错误提示

## 不在 v1 范围

- 频道归档（archive）功能 — 留 v2
- 恢复已删频道（undelete）
- 右键菜单删除入口 — v1 仅频道设置页面
- 批量删除频道
- 删除前数据导出
